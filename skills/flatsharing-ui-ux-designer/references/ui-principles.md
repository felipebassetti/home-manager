# UI Principles

## Product Tone

- parecer sistema operacional de negocio, nao landing page
- transmitir organizacao, clareza e confianca

## Good Screen Patterns

- cabecalho curto com acao primaria proxima
- toolbar compacta para filtros e contexto
- metricas resumidas em uma linha antes dos detalhes
- listas e tabelas com colunas legiveis e alinhamento consistente
- paineis laterais apenas quando ajudam a concluir a tarefa principal

## Anti-Patterns

- hero grande com pouco valor operacional
- card dentro de card sem necessidade
- espaco excessivo que empurra dados importantes para baixo
- muitas cores disputando atencao
- acoes primarias espalhadas em varios pontos da tela

## Responsive Expectations

- mobile deve manter prioridade de acao
- tabelas podem quebrar em blocos, mas sem perder rotulos
- filtros e formularios devem empilhar sem apertar campos
