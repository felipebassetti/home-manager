# Documentacao

Esta pasta concentra a documentacao operacional do projeto. 

## Estrutura

- `hus/`: historias de usuario que funcionam como cards, guia de desenvolvimento e base de teste

## Regra de uso

- Cada HU precisa ser pequena, testavel e orientada a valor de negocio
- Cada HU precisa ter criterios de aceite objetivos
- Cada HU precisa indicar status atual: `planned`, `in_progress`, `done`, `blocked`
- Cada HU deve citar telas, endpoints ou modulos impactados quando isso ajudar a execucao
- As HUs devem servir como backlog vivo do MVP
